## Overview

Restructured the LocalGround Toolkit from paste-and-run markdown prompts into a hybrid Node.js architecture. Deterministic filesystem operations (path-hash decoding, cloud detection, integrity checks, copy, seed, verify, scan) moved into `@localground/core`. The MCP server (`@localground/mcp`) exposes those operations as Claude Code tool calls; the standalone CLI (`@localground/cli`) provides direct terminal usage; five Claude Code skills (`/localground:seed`, `/localground:migrate`, `/localground:reap`, `/localground:cleanup`, `/localground:verify`) replace the v2.0.0 prompts as thin orchestrators with `allowed-tools` frontmatter. The v2.0.0 paste-and-run prompts remain untouched in `prompts/` as the no-install fallback.

6. **v2.0.0 prompts preserved as no-install fallback** — All five paste-and-run prompts in `prompts/` untouched, still functional. INFRA-06 satisfied; safety model invariant preserved across the v2 → v3 migration.

### Key Decisions

| Decision | Rationale | Outcome |
|---|---|---|
| npm workspaces (not Turborepo/Lerna/Nx) | Three packages with simple deps; abstractions would be premature | ✓ Validated — clean monorepo with cross-references working |
| MCP SDK v1.x (not v2) | Build on stable; v2 still pre-stable as of April 2026 | ✓ Validated — `@modelcontextprotocol/sdk@1.29.0` shipped without breakage |
| tsup for compilation (not Webpack/Rollup) | Single-file ESM bundles via esbuild; appropriate for Node CLI tools | ✓ Validated — clean builds, banner-injected shebangs work correctly |
| Vitest for testing (not Jest) | Native ESM, faster, integrates with tsup | ✓ Validated — 79-test suite runs in ~10s on CI |
| Bundle strategy Option A: `noExternal` core | Core stays private (not published); mcp + cli inline its source via tsup | ✓ Validated — verified at build level (zero remaining `from '@localground/core'` in dist); required moving core from `dependencies` to `devDependencies` to avoid phantom runtime dep |
| Manual first publish + OIDC subsequent | npm/cli#8544 — pre-publish trusted-publisher config not possible on npmjs.com | ✓ Pragmatic — accepts no provenance on v3.0.0 in exchange for getting v3.0.0 live; v3.0.1+ has full OIDC + provenance |
| Result type pattern (`Result<T, R>`) | Discriminated union; consumers narrow via `if (result.success)`; type system enforces error handling at every call site | ✓ Validated — no exceptions thrown from core, every error path checked |
| Stderr-only logging in MCP server | Stdout reserved for JSON-RPC transport; CRIT-1 invariant | ✓ Validated — `stdout-discipline.test.ts` enforces with 3 test cases |
| `--json` flag on every CLI command | Stdout = JSON in JSON mode; stderr = status, suppressed in JSON mode | ✓ Validated — flag works before or after subcommand, stderr empty in JSON mode |
| Real-fs test fixtures (not mocked) | Phase 14 found Windows reparse-point and OneDrive-path bugs that mocks cannot reproduce | ✓ Validated — `os.tmpdir()` + `fs.mkdtemp` + `afterEach` cleanup pattern caught real platform-specific edge cases |
| Filesystem-listing reverse-encode decoder | Sidesteps separator-guessing; any folder that physically exists decodes correctly regardless of mixed punctuation | ✓ Validated — closes UAT Defect B; OneDrive corporate paths decode correctly |
| Path-shape-only `looksLikeProject` predicate | No `.git`/`package.json` marker check; rejects root, home, <2 segments below home; explicit `--projects` bypasses | ✓ Validated — supports plain-folder projects (D-01) without polluting audit with root paths |

### Issues Resolved

- Decoder failed on mixed-punctuation OneDrive paths — closed by 14-08 (filesystem-listing reverse-encode)
- CLI/MCP detect surface returned `decodedPath: null` for all entries — closed by 14-09 (decode-and-enrich pattern at consumer surfaces; core `detect()` intentional null-return contract preserved)
- Audit auto-discovery treated `C:\` and `C:\Users\rlasalle` as projects — closed by 14-10 (`looksLikeProject` predicate)
- CLI copy and audit ran silently for minutes with no terminal feedback — closed by 14-11 (TIER 1 stderr status lines)
- Duplicate shebang in MCP dist — closed by 13-06 (tsup banner sole shebang source)
- `tsc --build` fails with `'incremental' has no effect when 'composite' is true` — closed in 12-01
- tsup DTS rollup fails with `composite: true` — closed in 12-02 (composite:false override per-package)
- `workspace:*` protocol unsupported in npm — closed in 12-01 (replaced with `"*"`)
- Phantom runtime dep on `@localground/core` would have bricked consumer installs — closed in 15-05 (moved to devDependencies)
- npm/cli#8544 chicken-and-egg with trusted publishers — worked around in 15-05 (manual first publish, then OIDC for subsequent)
