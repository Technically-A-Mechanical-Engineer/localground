# Quick Task 260630-m1i: Bump version 3.0.2 → 3.1.0 for the v3.1.0 release

<objective>
Align the npm/package version with the v3.1.0 GSD milestone by bumping every version-of-record from `3.0.2` to `3.1.0` and regenerating the lockfile. This is the pre-tag bump on known-green `master` (CI passed all 4 jobs — including the Phase-21 pinact gate — at commit `889f58e`). Per decision D-10, the bump lands post-CI-green and pre-tag. The Stage-3 `release.yml` preflight asserts `tag == mcp version == cli version`, so `@localground/mcp` and `@localground/cli` MUST both equal `3.1.0` or the publish blocks.
</objective>

<scope_correction>
Original Stage-2 scope named four manifests. A pre-edit `grep` for `3.0.2` surfaced a **fifth** version-of-record: `.claude-plugin/plugin.json` (the Claude Code plugin manifest), which has been kept in lockstep (currently `3.0.2`). Leaving it at `3.0.2` would make the repo internally inconsistent, so it is included.

`toolkitVersion` is **derived, not hardcoded** — `packages/mcp/src/index.ts` and `packages/cli/src/index.ts` read their version from `package.json` at runtime, and `seed()` takes `toolkitVersion` as a caller-supplied parameter. `scripts/verify-tarball.mjs` asserts the seed manifest's `toolkitVersion` equals `package.json`'s version (guarding the hardcoded-drift class). So the manifest bump propagates automatically — **no source edits**. The `toolkit_version: "2.0.0"` strings in `prompts/` and `docs/dev-status/dev-status-reap.md` are legacy v2.0.0 prompt-era artifacts and are intentionally untouched.
</scope_correction>
