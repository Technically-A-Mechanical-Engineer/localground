<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local tag ref → GitHub v* trigger → release.yml | pushing the tag is the single irreversible action; everything downstream publishes to the immutable registry |
| release job OIDC token → npm publish | short-lived id-token authorizes the publish; the trusted-publisher config (20-03) must match or auth fails |
| two independent publish HTTP transactions → registry | non-transactional; a mid-sequence failure yields an irreversible partial release |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-15 | Tampering | tagging a commit that has not passed CI | mitigate | D-10 STEP A blocks tag creation until the bump commit's own ci.yml run is green on all 3 OSes (headSha-matched). |
| T-20-16 | Spoofing | tag content not matching intended version/metadata | mitigate | D-10 STEP C `git show v3.0.1:packages/{mcp,cli}/package.json` verifies version 3.0.1 + repository present BEFORE the push; wrong content → delete tag, restart. |
| T-20-17 | Denial of Service (irreversible partial publish) | independent mcp/cli publish steps | mitigate | D-08 dry-run-both (in release.yml) runs before either real publish; Task 3 failure-handling documents the no-republish recovery (patch version only). Residual accepted: npm has no atomic multi-publish. |
| T-20-18 | Elevation of Privilege | OIDC id-token misuse / auth failure | mitigate | Pure-OIDC short-lived token (D-01); minimal permissions (20-02); failed auth publishes nothing and is retryable; token fallback is the documented emergency path, not the default. |
| T-20-19 | Repudiation | published artifact lacks provenance | mitigate | `--provenance` on both publishes + Node 22 (npm ≥11.5.1); provenance badge verified on npmjs.com in Plan 20-06 (SC3). |
</threat_model>
