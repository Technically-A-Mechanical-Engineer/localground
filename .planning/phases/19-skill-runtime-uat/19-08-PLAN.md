<objective>
Fix the root-cause defect discovered during 19-01/19-02: the five `/localground:*` skills are not invocable because they ship as loose `.claude/skills/*.md` files, which Claude Code does not register. Repackage them as a Claude Code PLUGIN ("localground") so `/localground:seed`, `/localground:migrate`, `/localground:reap`, `/localground:cleanup`, `/localground:verify` work as documented, then prove registration + routing with a REAL slash-command invocation.

This is a D-18 gap-closure sub-plan. It BLOCKS the re-run of UAT-01..05 (those validate skill invocation, which does not work until this lands). The MCP tool layer is already proven (19-01 seed manifest on disk; 19-02 copy/verify) — only the skill-invocation packaging is broken.

**Scope (H-4 decision, maintainer 2026-06-25):** Phase 19 validates that the plugin LOADS LOCALLY and routes (`claude --plugin-dir .` + the existing `--scope user` local-dist MCP server). The END-USER distribution path — installing the published plugin so its BUNDLED `.mcp.json` AUTO-REGISTERS the MCP server and routes for a fresh user — is OUT OF SCOPE here and moves to **Phase 20** (new plan + a new end-user-validation SC + PROJECT.md "two forms" -> "three forms" + DOC-03 plugin docs).

Design calls (locked with maintainer 2026-06-25):
- **D-A Location:** plugin at REPO ROOT (`.claude-plugin/plugin.json` + `skills/` at root). The repo IS the toolkit; simplest to load/test (`claude --plugin-dir .`) and coexists with `packages/`.
- **D-B MCP wiring (REVISED per review C-1):** UAT re-runs use the EXISTING `--scope user` local-dist registration (Phase 17/18-fixed code) ONLY. **19-08 ships NO plugin `.mcp.json`** — a plugin `.mcp.json` auto-starts on load and would spawn a second, competing `localground` server (npx unpublished/hangs or published-v3.0.0/unfixed) mid-UAT. The bundled `.mcp.json` is a Phase 20 deliverable, validated there as the end-user auto-register path (H-4).
- **D-C Distribution** (marketplace vs bundled-with-npm) + bundled `.mcp.json` + end-user auto-register validation: DEFERRED to Phase 20 (H-4). 19-08 only needs the structure correct and locally loadable.

Output: working `localground` plugin (manifest + 5 skills, no `.mcp.json`); dead loose-skill files removed under a guarded procedure; docs + tarball guard updated; `19-transcripts/plugin-registration.md`; honestly-corrected `19-UAT.md`. After this, 19-01 (seed) re-confirms via real command and 19-02..07 resume.
</objective>
