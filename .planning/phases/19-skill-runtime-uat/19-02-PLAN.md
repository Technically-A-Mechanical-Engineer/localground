From `.claude/skills/localground-migrate.md` (lines 17-21 — the SESSION DETECTION logic, two branches NOT three):
```
if (state file exists AND state.session === 1) → Session 2
else                                            → Session 1
```

So the THREE input states map to TWO code branches:
| Input state                  | Branch taken | What runs                                                          |
|------------------------------|--------------|--------------------------------------------------------------------|
| No state file present        | Session 1    | detect → copy → verify → write state                                |
| state.session === 1          | Session 2    | read state → migrate settings → update state.session = 2           |
| state.session === 2          | Session 1    | detect → copy → verify → write state (no early-exit)                |

State file location (L-7): written to `<destinationBasePath>/localground-migrate-state.json` — the destination BASE path, NOT inside any project subdirectory.

State file schema (Session 1 post-write): `{ version, session: 1, timestamp, sourcePath, destinationPath, projects: [{name, sourcePath, destinationPath, verification: {success, checksumMatch, gitTagPresent, commitHashMatch}}] }`

State file schema (Session 2 post-update): adds `completedTimestamp`, `session: 2`, per-project `settingsMigrated`, `referencesUpdated`.

Continuation-token loop (skill step 2 in Session 1): the skill calls `localground_copy` with `source` + `target` (no token first call); if response `done: false`, extract `token`, call again with same source/target/token. Repeats until `done: true`.
