<objective>
Restore `tsc --build` as a CI quality gate covering both source AND test files (TEST-01), without weakening strict mode. The naive approach — adding `test/**/*` to each `packages/*/tsconfig.json` `include` array — is rejected because the per-package composite tsconfigs use `rootDir: "./src"` and `composite: true`, which TypeScript enforces as "all included files must be under rootDir". A separate root-level `tsconfig.test.json` with `composite: false, noEmit: true` is the chosen escape hatch.

Sequence: (1) gate-scope expansion creates the new tsconfig and updates the script — at this point tsc errors surface; (2) fix all surfaced errors file-by-file with explicit type annotations, in dependency order (core → mcp → cli) per D-Claude-3, committing per logical cluster; (3) wire the CI step in `.github/workflows/ci.yml` per D-03/D-04 only AFTER errors are fixed, so the first push under the new gate is green.

Purpose: Restore the structural typing protection that v3.0.0 dropped (D-18). With this in place, future codebase changes (Phase 17 CORE-13/14 regex calibration, Phase 19 UAT discoveries) land under a hardened typing gate.

Output: New `tsconfig.test.json`, updated `package.json` script, surfaced errors fixed with explicit annotations, new CI step in `.github/workflows/ci.yml`.
</objective>
