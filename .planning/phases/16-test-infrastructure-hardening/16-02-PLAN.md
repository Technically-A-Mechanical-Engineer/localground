<objective>
Eliminate the Vitest cleanup hang on `npm test` exit (TEST-02) by adding describe-scoped `afterEach` reapers to both MCP and CLI smoke test files. The hang exists today because spawned child processes (MCP server child via `spawnServer()`, CLI child via `runCli()`) are killed inside per-test `try/finally` blocks but their `exit` events are never awaited — Vitest's worker shutdown waits for the open handles, producing the multi-second hang the developer currently `Ctrl+C`s.

Purpose: `npm test` exits with the same code Vitest reports, on all three OS targets, with no manual Ctrl+C. Restores the CI gate's reliability for TEST-01's first `tsc + tsup + vitest` triplet.

Output: Two test files with `afterEach` cleanup hooks; `npm test` exits cleanly; no config files touched (per D-06); no production code touched.
</objective>

Phase 16-02 modifies the parent-child trust boundary in test files only. The MCP server child receives JSON-RPC frames over stdin (untrusted in production, controlled in tests); the CLI child receives argv.
